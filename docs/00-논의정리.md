# 논의 정리 (2026-01-29)

## 제품 구성
- **모바일 앱**: Android / iOS, Flutter로 개발
- **웨어러블**: Apple Watch + Galaxy Watch(Wear OS 가정), 각 플랫폼 네이티브로 개발
- **게임 형태**: 1:1 진행, 양쪽(A/B)이 점수를 올리고/내리고/초기화 가능해야 함

## 웨어러블 최소 기능
- **점수 +1 / -1**
- **A/B(좌/우) 선택** 또는 A/B 별 버튼 제공
- **초기화(리셋)**: 오동작 방지를 위해 “길게 누르기/확인” 같은 UX 권장
- **현재 점수 표시**

## 동기화 기본 전제(권장안)
- **모바일이 Authority(정답 점수)** 를 가진다.
- 워치는 **명령(command)** 을 폰에 전송한다.
- 폰은 command를 적용해 점수를 갱신하고, **최신 상태(state)** 를 워치로 다시 전송한다.

## 결정된 기술 방향
- 모바일: **Flutter**
- 워치: **플랫폼 네이티브**
  - Apple Watch: WatchConnectivity 기반
  - Wear OS: Data Layer(MessageClient/DataClient) 기반

## 아직 확정이 필요한 항목(질문)
- “한 대의 폰 + 양쪽 워치”인지, “각자 폰/워치”인지(Authority/충돌 전략에 영향)
- 워치 단독(폰 분리) 카운팅이 필요한지 여부(오프라인 큐/충돌 처리에 영향)
- 갤럭시워치 범위가 Wear OS만인지(Tizen 포함 여부)
- 점수 규칙(단순 점수 vs 세트/듀스 등)
- 저장/공유 범위(로컬만 vs 계정/클라우드/공유)
