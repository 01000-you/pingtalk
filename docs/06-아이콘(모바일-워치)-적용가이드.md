# 아이콘(모바일 + Wear OS) 적용 가이드

## 왜 아이콘이 2개가 필요하나?
- **모바일 앱 아이콘**: Android/iOS 런처/스토어/설치 화면에 표시
- **워치 앱 아이콘**: Wear OS 앱 목록/타일/설치 목록에 표시

권장: **한 장의 “원본 아이콘(1024×1024 PNG)”** 를 만들고, 각 플랫폼 아이콘을 자동 생성합니다.

---

## 1) 원본 아이콘 준비
- 파일: `mobile/assets/icon/app.png` (PNG)
- 크기: 1024×1024
- 배경 포함(투명도 가능하지만, Adaptive icon 대응을 위해 배경이 있는 버전도 권장)

> 원본 파일은 직접 디자인하거나, 이후 단계에서 디자이너/브랜딩 작업으로 교체해도 됩니다.

---

## 2) Flutter(모바일) 아이콘 자동 생성
Flutter는 여러 플랫폼 아이콘 사이즈가 많아서 수동 교체보다 **생성 도구**를 권장합니다.

### 설정
- 패키지: `flutter_launcher_icons`
- 생성 명령:
```bash
cd mobile
flutter pub get
flutter pub run flutter_launcher_icons
```

### 참고
- Android/iOS 아이콘이 자동으로 교체됩니다.
- 커밋에는 생성 결과물(플랫폼별 아이콘)이 포함되어도 되지만, 팀 규칙에 따라 선택하세요.

---

## 3) Wear OS 아이콘 적용(현재 프로젝트 구조 기준)
Wear OS 앱은 `mobile/android/wear` 모듈에 있습니다.

### 권장 방식(Android Studio로 1회 생성)
1. Android Studio에서 `mobile/android` 열기
2. `wear` 모듈 선택
3. `wear/src/main/res` 우클릭 → **New → Image Asset**
4. Launcher Icons 생성(Adaptive/Legacy)
5. `wear/src/main/AndroidManifest.xml`의 `<application android:icon="...">`에 적용

> Wear OS도 Android와 동일하게 launcher icon 리소스가 필요합니다.

---

## 4) 스토어 제출용 그래픽(필수/권장)
- **Play Store**
  - 앱 아이콘(자동 생성된 런처 아이콘과 별개로 스토어 리스팅용)
  - Feature graphic, 스크린샷(폰/워치)
- **App Store**
  - 아이콘/스크린샷/미리보기
